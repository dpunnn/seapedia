# SEAPEDIA — UI/UX Design Brief (Complete)

## Tentang SEAPEDIA
SEAPEDIA adalah marketplace e-commerce multi-peran (Seller, Buyer, Driver, Admin) yang menjadi tugas seleksi Software Engineering Academy COMPFEST 18 Fasilkom UI. Aplikasi ini perlu tampil profesional, modern, dan terpercaya layaknya marketplace nyata. Semua halaman harus menggunakan komponen dan sistem desain yang konsisten.

---

## Brand Identity

**Nama**: SEAPEDIA
**Tagline**: "Marketplace Terpercaya untuk Semua"


## Order Status Color System
- SEDANG_DIKEMAS: Amber (kuning) `#D97706`
- MENUNGGU_PENGIRIM: Biru `#2563EB`
- SEDANG_DIKIRIM: Oranye `#EA580C`
- PESANAN_SELESAI: Hijau `#16A34A`
- DIKEMBALIKAN: Merah `#DC2626`

---

## Layout System

### Navbar (semua halaman publik)
- Sticky top, height 64px, background putih, shadow subtle
- Kiri: Logo "SEAPEDIA" dengan ikon belanja kecil, biru bold
- Tengah: Link navigasi (Produk, Ulasan) — tersembunyi di mobile
- Kanan (Guest): Tombol "Masuk" (outline) + "Daftar" (solid primary)
- Kanan (Login): Badge role aktif berwarna sesuai role, avatar dengan inisial email, dropdown menu (Dashboard, Ganti Role, Keluar)
- Mobile: hamburger menu

### Dashboard Sidebar Layout
- Sidebar kiri 240px, fixed/sticky
- Header sidebar: logo SEAPEDIA kecil + nama user + badge role aktif berwarna
- Nav items dengan ikon (lucide-react) + label
- Item aktif: background primary light, text primary, border-left 3px primary
- Item hover: background gray-50
- Footer sidebar: tombol "Ganti Role" + "Keluar"
- Main content: padding 24px, background `#F8FAFC`
- Mobile: sidebar jadi bottom sheet atau hamburger

### Sidebar Nav per Role:
- **Buyer**: Dashboard, Dompet, Alamat, Keranjang, Pesanan, Laporan
- **Seller**: Dashboard, Toko Saya, Produk, Pesanan, Pendapatan
- **Driver**: Dashboard, Cari Pekerjaan, Pekerjaan Aktif, Riwayat
- **Admin**: Dashboard, Pesanan, Overdue, Pengguna, Toko, Produk, Pengiriman, Voucher, Promo, Simulasi Waktu

---

## Komponen Global

### ProductCard
- Gambar produk (aspect-square, object-cover, bg-gray-100 fallback paket emoji)
- Nama produk (2 baris clamp, font-medium)
- Harga (biru bold)
- Nama toko (abu-abu kecil)
- Badge stok (hijau kalau ada, merah kalau habis)
- Hover: shadow naik, slight scale

### StatusBadge
- Pill shape, padding kecil, sesuai warna order status di atas
- Teks status dalam Bahasa Indonesia

### OrderTimeline (5 langkah)
- Vertical stepper: Sedang Dikemas → Menunggu Pengirim → Sedang Dikirim → Pesanan Selesai / Dikembalikan
- Step yang sudah lewat: ikon centang biru, garis solid
- Step aktif: ikon dot biru beranimasi pulse
- Step belum: ikon lingkaran abu-abu, garis putus

### PriceBreakdown
- List item: label kiri, nilai kanan
- Diskon: teks hijau dengan tanda "-"
- PPN: label dengan tooltip "12% dari (subtotal - diskon)"
- Separator sebelum Total
- Total: bold, lebih besar

## 12. Dashboard Buyer — Keranjang & Checkout (`/dashboard/buyer/cart`)

**Tujuan:** Review keranjang dan lanjut ke checkout. Halaman paling kompleks buyer.

**Layout 2 kolom (desktop): Kiri = Items, Kanan = Summary**

### Kolom Kiri — Isi Keranjang

**Banner Single-Store Rule:**

**List Item:**
- Per item: gambar produk kecil (60×60), nama, harga satuan
- Input quantity (−/angka/+) dengan update otomatis
- Tombol hapus (ikon trash, merah hover)
- Subtotal per item (kanan)

**Tombol Kosongkan Keranjang** (merah kecil, bottom left)

**Empty State:**
- Ilustrasi keranjang kosong
- "Keranjangmu masih kosong"
- Tombol "Mulai Belanja" → /products

### Kolom Kanan — Ringkasan & Checkout

**Pilih Alamat:**
- Dropdown/select, tampilkan label + kota
- Link "Tambah Alamat Baru"

**Pilih Metode Pengiriman:**
- 3 opsi radio card:
  - INSTANT: "Pengiriman Instan — Rp 25.000" + sublabel "Estimasi hari ini"
  - NEXT_DAY: "Pengiriman Besok — Rp 15.000"
  - REGULAR: "Reguler — Rp 10.000" + sublabel "3-5 hari kerja"

**Tombol "Konfirmasi Pesanan":**

## CATATAN TEKNIS UNTUK DESAINER

1. **Framework**: Next.js App Router — desain harus mengasumsikan client-side rendering, bisa pakai animasi/transisi
2. **Komponen**: Shadcn/ui sebagai base — button, input, card, badge, select, dialog, separator sudah tersedia
3. **Icons**: Lucide React — set ikon modern yang bersih
4. **Toast**: Sonner — notifikasi muncul di pojok kanan bawah
5. **Loading states**: Semua fetch data harus ada skeleton loading
6. **Empty states**: Setiap list/grid harus ada empty state yang informatif
7. **Error states**: Form error muncul di bawah field, merah, teks kecil
8. **Mobile responsive**: Semua halaman harus responsif — sidebar jadi bottom nav atau hamburger di mobile
9. **Accessibility**: Contrast ratio minimal 4.5:1, semua interactive element punya focus ring
